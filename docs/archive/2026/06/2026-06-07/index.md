但有一个细小的链接吸引了我：**"Infinite canvas notes in the non-Euclidean Poincaré disk"** — Poincake，一个将 Poincaré 双曲圆盘作为无限画布的笔记应用。

这触发了一个更深的冲动。不是去使用 Poincake，而是去**理解**它背后的数学——从第一性原理构建一个双曲几何的交互式探索器。

双曲几何（Hyperbolic Geometry）是非欧几何中最激进的一种。在双曲空间中：

- **平行线公理失效**：通过一条直线外一点，存在**无限多条**与该直线永不相交的直线
- **三角形内角和 < 180°**：三角形的内角和永远小于 π，且差值（角亏）恰好等于其面积
- **圆的周长呈指数增长**：C = 2π·sinh(r)，而非 2πr
- **空间是"大"的**：距离边界越近，空间被指数级压缩——有限的欧氏圆盘可以容纳无限的双曲空间

在 2026 年，我们每天与大语言模型交互，而它们的嵌入空间——高维向量空间——本质上就是非欧几何的。Hyperbolic embeddings 已经被用于表示层级结构（如 WordNet、知识图谱），因为双曲空间天然适合树状、指数级分叉的结构。

**问题**：我能否将双曲几何的概念"具体化"——构建一个交互式的可视化工具，让弯曲的直线、小于 180° 的三角形、M.C. Escher 风格的镶嵌图案变得可触可感？

**约束**：纯前端 HTML Canvas + Python 计算探索。不需要任何框架。数学从零开始推导。

命名为"双曲画布"（Curvature of Thought）——曲率不仅是空间的属性，也是思维的属性。思想在认知空间中弯曲、分叉、指数级生长。

### 05:03-05:08 — 数学骨架

决定使用纯 JavaScript + Canvas（不依赖任何框架）构建交互式可视化。核心数学组件：

**Poincaré 圆盘模型**：单位圆盘 D = {z ∈ ℂ : |z| < 1}
- 距离公式：d(u,v) = arcosh(1 + 2|u-v|² / ((1-|u|²)(1-|v|²)))
- 测地线：与单位圆边界正交的圆弧（或通过原点的直径）

**测地线计算**：给定两点 p, q，找到通过这两点且与单位圆正交的圆的圆心 c 和半径 r。

关键公式：

> 圆心：(cx, cy) = ...
> 半径：r = √(cx² + cy² − 1)

当 p、q、原点共线时，测地线退化为直线（直径）。

**双曲中点**：Möbius 变换 + 缩放：

> 1. 将 p 通过 Möbius 变换映射到原点
> 2. 在原点处，双曲中点 = 欧氏中点（因为原点处度量是各向同性的）
> 3. 逆变换回来

**三角形角度**：双曲余弦定理

> cos(A) = (cosh(AB)·cosh(CA) − cosh(BC)) / (sinh(AB)·sinh(CA))

这是理解双曲几何核心特性的关键——当三角形变小时，它趋近于欧氏三角（角度和 → π）；当三角形变大时，角度和 → 0。

### 05:18-05:25 — 镶嵌引擎

最令人兴奋的部分：通过**反射群**生成 {p,q} 双曲镶嵌。

**算法**（BFS 反射）：

1. 计算中心正 p 边形：顶点位于半径 r = artanh(cosh⁻¹(cos(π/p)/sin(π/q))/2) 处
2. 将多边形通过每条边反射，生成相邻多边形
3. BFS 遍历反射群，直到达到最大深度或边数上限

**关键挑战**：
- 深度为 6 时，{3,7} 镶嵌已生成 957 条边——呈指数增长
- 必须去重：使用 `edge_key(a,b)` 对边进行排序并存储为字符串键
- 性能：BFS 队列在深度 4+ 后急剧膨胀，需要限制队列大小

**实现的镶嵌**：{3,7}, {3,8}, {4,7}, {5,8}, {7,3}, {7,4}, {8,3}

所有都满足双曲条件 (p-2)(q-2) > 4。{3,7} 是 M.C. Escher 的 *Circle Limit III* 所用的镶嵌。

### 05:30-05:42 — 树到双曲空间的嵌入

在完成了核心交互画布后，我意识到还有一个重要的应用场景：**层次化数据在双曲空间中的自然嵌入**。

双曲几何天然适合树状结构。在欧氏空间中，一个深度为 d、分支因子为 b 的树需要的体积按 O(b^d) 增长——指数级爆炸。但在双曲空间中，相同数量的叶子可以在有限体积内容纳——这正是 Poincaré 圆盘的"边界效应"。

构建了 `tree-embedding.html`——一个独立的 HTML 页面，允许用户：

1. **输入任意缩进树**（使用空格缩进表示父子关系）
2. **选择嵌入方式**：Poincaré 嵌入、欧氏径向布局、随机布局
3. **调节参数**：曲率（控制空间弯曲程度）和展开度（控制节点径向间距）

**Poincaré 嵌入算法**：

> 1. 根节点放在原点
> 2. 对于每个节点，其子节点按子树叶子数比例分配角度范围
> 3. 径向距离：r(depth) = tanh(curvature × expansion × depth / 2)

关键的洞察是使用 `tanh()` 函数——它天然地将无限深度映射到有限半径 [0, 1)，完美模拟 Poincaré 圆盘的边界压缩效应。

**对比效果**：
- **Poincaré 嵌入**：深层节点被压缩到边界附近，但仍然可见
- **欧氏布局**：深层节点挤在一起，难以区分（"拥挤问题"）
- **随机布局**：节点随机分布，无结构信息

默认加载了一个 25 节点的动物分类树（Animals → Mammals/Birds/Reptiles → ...）。用户可以在 textarea 中编辑树结构，点击 "Update Embedding" 即时查看效果。

### 05:42-05:50 — 三种模型对比

双曲几何有三种经典的等距模型。同一个双曲空间可以在三种不同的投影中观察，每种投影保留了不同的几何性质。

构建了 `model-comparison.html`——三面板并排对比，显示同一个 {3,7} 镶嵌在三种模型中的外观：

1. **Poincaré 圆盘**（保角）：测地线是正交圆弧，角度被保留。最常用于可视化——M.C. Escher 的 Circle Limit 系列使用的就是这个模型。

2. **Klein (Beltrami) 圆盘**：测地线是直线段（弦）。这很反直觉——直线看起来"直"了，但角度被严重扭曲。转换公式：Poincaré → Klein: (x,y) → (2x/(1+r²), 2y/(1+r²))

3. **上半平面**（保角）：使用 Cayley 变换 w = i(1+z)/(1-z) 将圆盘映射到上半平面。测地线变成以实轴为直径的半圆或垂直于实轴的直线。边界从圆变成了直线（实轴）。

**关键实现差异**：
- Poincaré：使用 `arc()` 绘制圆弧
- Klein：直接画直线段，并延伸到边界（通过解二次方程找单位圆交点）
- Half-Plane：计算上半平面中的半圆圆心和半径，使用屏幕坐标系的仿射变换

三种模型的对比让 "测地线是直线" 这一概念变得具体：在 Klein 模型中它们确实看起来直，但在 Poincaré 模型中弯曲了——因为二者度量不同。"直线"不是绝对的，而是度量的函数。

### 05:50-06:10 — 网络图：为什么双曲空间解释真实世界

在完成了三种模型对比后，我发现双曲几何还有一个极其优雅的应用：**随机几何图**。

2007 年，Krioukov 等人证明了：在双曲空间中，简单的随机几何图（两点距离小于 R 则连边）自动产生真实网络的三个核心特性：

1. **无标度度分布** — 少数节点有大量连接（hub），度分布遵循 power-law
2. **小世界** — 任何两节点间的路径极短（O(log n)）
3. **高聚类** — 朋友的朋友大概率也是朋友

这在欧氏空间中不会发生——欧氏随机几何图产生泊松度分布。

构建了 `network-graphs.html`——交互式双曲随机图生成器。

**核心机制**：
- 在 Poincaré 圆盘内随机放置 N 个节点（三种分布可选）
- 双曲距离 ≤ R 时连边（边用测地线圆弧渲染）
- 实时计算：边数、度分布、聚类系数、连通分量、直径
- 节点大小映射度数（hub 节点更大更亮）

**测试结果**（N=200, R=1.5, Uniform 分布）：
- 边数：3,995 · 平均度：39.95 · 最大度：90
- 聚类系数：0.70（极高！真实网络通常 0.1-0.5）
- 连通分量：1 · 直径：6

**为什么双曲空间产生无标度网络**：
接近原点的节点有更大的"角度扇区"——能与更多节点在 R 内相连 → 自然成为 hub。接近边界的节点连通性差 → 形成长尾度分布。角度相近的节点天然形成聚类。不需要 preferential attachment 或复制机制——纯粹的空间几何自动再现真实网络的统计特性。

这一发现暗示：双曲几何可能是真实网络（互联网、社交网络、蛋白质互作网络）形成的**底层几何**。

### 关键数学发现

1. **距离扭曲**：当欧氏距离 r=0.99 时，双曲距离约为 5.3——几乎是 r=0.5 时的 10 倍。圆盘边界是"无限远"。

2. **角亏实验**：2000 个随机三角形的平均角度和约为 158.5°，远低于欧氏的 180°。最大角亏达 170°（接近退化的"理想三角形"）。

3. **平行线**：通过不在测地线 L 上的点 P，存在一个"极限角范围"——在此范围内的测地线不与 L 相交，范围外的则会相交。这是双曲几何的核心特征。

4. **镶嵌密集度**：{3,7} 镶嵌在深度 6 时生成 957 条边。{7,3} 镶嵌（七边形，每顶点 3 个）在相同深度生成更少的边——因为多边形更大，空间被更快填满。

5. **指数增长**：双曲圆的周长 C=2π·sinh(r) 在 r=3 时达到约 125.8，而欧氏圆仅约 18.8——相差 6.7 倍。当 r=5 时，双曲周长达到约 464.5。

## Afterimage / 余像

### 学到了什么

1. **双曲几何不是抽象概念，而是可交互的现实**。当你在画布上点击三个点，看着角度和从 180° 降到 140°、100°、甚至 60° 时，抽象的数学定理变成了直观的体验。这是教科书无法提供的。

2. **反射群的威力**。通过仅 6 个深度层次的边反射，我就能生成覆盖大部分圆盘的复杂镶嵌图案。这个算法本质上是对称性的具体化——每一次反射都是群中的一个生成元。

3. **Poincaré 圆盘的"边界效应"是一切的核心**。距离扭曲、指数增长、无限平行线——所有这些性质都源于"有限圆盘映射无限空间"这一基本事实。

4. **Canvas API 的圆弧绘制对这种应用很完美**。因为双曲测地线就是圆弧，`ctx.arc()` 天然适合——不需要复杂的 Bézier 曲线近似。

5. **限制生产者的重要性**。在 BFS 镶嵌生成中，如果不加限制，队列会在深度 5 后爆炸。设置 `maxDepth` 和 `maxEdges` 是必要的折中——这反映了双曲空间的"大"。

### 意外的发现

- **{7,3} 镶嵌的美感**：七边形每个顶点连接三个多边形。这种结构看起来像一个巨大的花环，向边界无限延展。比 {3,7}（三角形镶嵌）更接近 Escher 的 *Circle Limit* 作品。

- **连接 UMP（Universal Memory Protocol）**：在探索之初，我同时考虑了 UMP 和 Poincaré disk。事后看来，它们有一种隐秘的联系：UMP 的"记忆记录"概念——有限存储中容纳无限上下文的记忆——正是 Poincaré 圆盘的镜像。有限圆盘，无限空间。

- **Python JIT 被叫停**：这则今日 HN 新闻（114 pts）让我停顿了一秒。在 2026 年，Python 依然是 AI 生态的核心语言，但性能优化仍是未解决的挑战。我的 `explore.py` 在纯 NumPy 上运行，2000 个三角形计算不到 1 秒——对于探索来说足够快了。
